Part 2 Min-Max Normalization — Making Numbers Fair
🏃
Real-life analogy: Imagine comparing a student's age (18) with their marks (95) — totally different scales! If you feed this to a model, the model thinks "age=18" is more important than "marks=9" just because of the big number. Normalization squishes ALL columns into the same scale: 0 to 1. Now everything is fair!
Formula: (value - minimum) ÷ (maximum - minimum)
❓
WHY IS THIS CODE HERE?

ML models like KNN or Neural Networks get confused when features have very different ranges. Normalization makes every feature contribute equally. This is one of the most important preprocessing steps in all of data science.

📐 PART 2 — Normalize all numeric columns to [0, 1]

Part 3 Label Encoding — Giving Numbers to Text
🏷️
Analogy: Imagine your school has 3 houses — Red, Blue, Green. The computer can't work with words, so you assign: Red=0, Blue=1, Green=2. That's Label Encoding! Each category gets a unique integer.

Part 4 One-Hot Encoding — A Better Way for Categories
🎯
Problem with Label Encoding: When we say setosa=0, versicolor=1, virginica=2, the model might think virginica (2) is GREATER than setosa (0). But species have NO order!

One-Hot Encoding Fix: Create 3 new columns — one per species. Each row gets a 1 in its species column, 0 everywhere else. No false ordering!
setosa → [1, 0, 0]   versicolor → [0, 1, 0]   virginica → [0, 0, 1]
🎯 PART 4 — One-Hot Encode species into 3 binary columns
species = df[['species']]
# ↑ Double brackets [[ ]] = keeps it as a DataFrame (2D), needed by OneHotEncoder
#   Single bracket [ ] gives a Series (1D) which OHE doesn't accept

encoder = preprocessing.OneHotEncoder()

encoded_species = encoder.fit_transform(species).toarray()
# ↑ fit_transform → creates binary matrix, .toarray() converts sparse to normal array

encoded_df = pd.DataFrame(encoded_species, columns=encoder.categories_[0])
# ↑ Wraps the array in a DataFrame. Column names = the actual category values

df_encoded = pd.concat([df, encoded_df], axis=1)
# ↑ axis=1 means "paste side-by-side" (column-wise). Joins original + encoded columns
df[['species']]
📋 Double square brackets ensure the result stays as a 2D table (DataFrame), not a 1D list. OneHotEncoder requires 2D input.
OneHotEncoder()
⬜ Creates separate binary (0 or 1) columns for each unique category. Each flower gets a 1 in its species column and 0 in all others.
.toarray()
🗃️ The encoder returns a "sparse matrix" (memory-efficient but hard to read). .toarray() converts it to a regular easy-to-use NumPy array.
encoder.categories_[0]
🏷️ Gets the category names from the encoder. Used as column headers so columns are named "Iris-setosa", "Iris-versicolor", "Iris-virginica" — not just 0, 1, 2.
pd.concat([df, encoded_df], axis=1)
🔗 axis=1 = join tables side by side (add columns). If axis=0, it stacks rows. We're appending the 3 new OHE columns to the right of the original table.

Part A Simple Linear Regression using NumPy
📏
What is a regression line? You have dots on a graph (student scores). You want to draw ONE straight line that passes as close as possible to ALL the dots. That line has the equation: y = m × x + c where m = slope (steepness) and c = intercept (where it hits the Y-axis). Once you have this line, you can PREDICT y for any new x!
📐 PART A — Fit a line to 5 students' marks and make predictions
x = np.array([95, 85, 80, 70, 60])   # subject A marks (input)
y = np.array([85, 95, 70, 65, 70])   # subject B marks (output to predict)

model = np.polyfit(x, y, 1)
# ↑ Finds the BEST FIT LINE through these 5 points
#   Returns: [slope, intercept] = the m and c in y = mx + c
#   The "1" means degree 1 = straight line (not a curve)

predict = np.poly1d(model)
# ↑ Converts [slope, intercept] into a callable function
#   Now you can do predict(65) to get the predicted y for x=65

predict(65)   # What mark in subject B for a student with 65 in subject A?

y_pred = predict(x)   # Predicted y for all 5 training points

from sklearn.metrics import r2_score
r2_score(y, y_pred)
# ↑ R² score: How good is our line? 1.0 = perfect, 0 = useless

# Draw the regression line
y_line = model[1] + model[0] * x   # y = intercept + slope × x
plt.plot(x, y_line, c='r')          # draw the red line
plt.scatter(x, y, c='r')            # plot actual data points
np.array([95, 85, ...])
📦 Creates a NumPy array — like a Python list but much faster for math. x = scores in subject A for 5 students. y = their scores in subject B.
np.polyfit(x, y, 1)
📐 Uses Least Squares method to find the line y=mx+c that minimizes total distance from all points. Returns [m (slope), c (intercept)]. The "1" = degree 1 = straight line.
np.poly1d(model)
🔧 Converts the [slope, intercept] array into a usable function. Instead of manually computing m*x+c every time, you just call predict(65).
r2_score(y, y_pred)
⭐ R² (R-squared) tells how well the line explains the data variation. 1.0 = line fits perfectly. 0 = line is no better than just predicting the mean every time.
model[1] + model[0] * x
📊 Manually computing y = intercept + slope × x. model[0]=slope, model[1]=intercept. Used to get the y-values along the line for plotting.
Part B Multiple Linear Regression — California Housing Dataset
🏠
WHY IS THIS CODE HERE?

In Part A, we used just ONE feature (subject A marks) to predict ONE output. In real life, house prices depend on MANY factors — number of rooms, location, income of area, age of house, etc. Multiple Linear Regression handles many input features simultaneously to predict one output (price).

🏠 PART B — Predict house prices from 8 features using Linear Regression
from sklearn.datasets import fetch_california_housing
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error

housing = fetch_california_housing()
# ↑ Loads a built-in dataset of 20,000+ California homes
#   Features: avg rooms, avg bedrooms, population, income, etc.
#   Target: median house price (in $100,000s)

data = pd.DataFrame(housing.data)
data.columns = housing.feature_names
data['PRICE'] = housing.target
# ↑ Build the full table: 8 feature columns + 1 PRICE column

data.isnull().sum()
# ↑ Always check for missing values before training. All zeros = clean data!

x = data.drop(['PRICE'], axis=1)  # features (all columns except price)
y = data['PRICE']                  # target (what we want to predict)

xtrain, xtest, ytrain, ytest = train_test_split(x, y, test_size=0.2, random_state=0)
# ↑ Split: 80% data for training, 20% kept aside for testing
#   random_state=0 = always same split (reproducible)

lm = LinearRegression()
lm.fit(xtrain, ytrain)
# ↑ TRAINING: model learns coefficients (weights) for each feature
#   It finds the formula: price = w1×income + w2×rooms + w3×age + ...

ytrain_pred = lm.predict(xtrain)
ytest_pred  = lm.predict(xtest)
# ↑ Use the trained model to predict prices for both train and test sets

mse = mean_squared_error(ytest, ytest_pred)
print(mse)
# ↑ MSE = average of (actual price - predicted price)²
#   Lower MSE = better model
fetch_california_housing()
🏘️ A ready-made dataset inside sklearn. Contains info about California houses. No need to download a CSV — sklearn provides it directly. Has 8 input features and 1 price target.
data.drop(['PRICE'], axis=1)
✂️ We can't use PRICE as a feature to predict PRICE — that's cheating! axis=1 = drop a column (not a row). Creates the feature matrix x without the target column.
train_test_split(x, y, test_size=0.2)
🔀 Randomly shuffles and splits data: 80% to train, 20% to test. Like studying from 80% of past papers, then testing yourself on the remaining 20% you haven't seen.
lm.fit(xtrain, ytrain)
🎓 This is where learning happens. The model studies the training data and finds the best weights (coefficients) for each feature. Like a student cramming before an exam.
lm.predict(xtest)
🔮 Uses the learned weights to predict prices for NEW (test) houses the model never saw during training. Tests if it truly learned or just memorized.
mean_squared_error(ytest, ytest_pred)
📉 Compares actual prices vs predicted prices. Squares the difference (so positive/negative errors don't cancel), then takes the average. Lower = better predictions.

Your job: Train a Logistic Regression model to classify customers, then evaluate HOW GOOD the model is using a Confusion Matrix and 5 metrics: Accuracy, Precision, Recall, F1 Score, and Specificity. Finally, take user input and predict their class.

Theory Understanding the Confusion Matrix (MOST IMPORTANT!)
🩺
Medical test analogy: You test 100 people for a disease. Some actually HAVE it (Positive), some don't (Negative). Your test might be right or wrong. That gives 4 possible outcomes:
• True Positive (TP): Patient HAS disease, test says YES ✅
• True Negative (TN): Patient has NO disease, test says NO ✅
• False Positive (FP): Patient has NO disease, but test says YES ❌ (False Alarm!)
• False Negative (FN): Patient HAS disease, but test says NO ❌ (Missed Case!)
Model Predicted: YES (1) Model Predicted: NO (0)
Actual: YES (1) ✅ True Positive (TP)
Correctly said YES		 ❌ False Negative (FN)
Said NO, but was YES
"Missed it!"
Actual: NO (0) ❌ False Positive (FP)
Said YES, but was NO
"False alarm!"		 ✅ True Negative (TN)
Correctly said NO
Accuracy
(TP + TN) × 100 / Total
Overall % correct. But misleading if classes are imbalanced!
Precision
TP / (TP + FP)
Of all we said YES, how many were actually YES? Avoids false alarms.
Recall (Sensitivity)
TP / (TP + FN)
Of all actual YES, how many did we catch? Avoids missing cases.
F1 Score
2 × P × R / (P + R)
Harmonic mean of Precision & Recall. Best single metric overall.
Specificity
TN / (TN + FP)
Of all actual NO cases, how many did we correctly identify as NO?
Part 1 Load Data & Prepare Features
🧹
WHY ARE WE DROPPING 'Genre' AND SELECTING LAST COLUMN?

The 'Genre' column is text (Male/Female) — we'd need to encode it separately, and the assignment skips it for simplicity. The last column in the dataset is the target variable (0 or 1 = will customer buy or not). All other columns are input features the model learns from.

🧹 PART 1 — Load, clean and split features from target
import pandas as pd, numpy as np
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import confusion_matrix

df = pd.read_csv('data5.csv')

df1 = df.drop("Genre", axis=1)
# ↑ Remove the 'Genre' (gender) column because it's text
#   and we're not encoding it in this assignment. axis=1 = drop column.

y = df1.iloc[:, -1].values
# ↑ iloc[:, -1] = ALL rows, LAST column = the target (0 or 1)
#   .values = convert to NumPy array (required by sklearn)

X = df1.iloc[:, :-1].values
# ↑ iloc[:, :-1] = ALL rows, ALL columns EXCEPT the last = the features
#   (Age, Annual Income, Spending Score — the inputs)
df.drop("Genre", axis=1)
🗑️ Removes the Genre column. axis=1 = remove a column (axis=0 would remove a row). The result is stored in df1 — the original df is unchanged.
df1.iloc[:, -1].values
🎯 -1 means "the last column" (Python negative indexing). This is the target variable — what we're trying to predict (0=no purchase, 1=purchase). .values gives a plain NumPy array.
df1.iloc[:, :-1].values
📋 :-1 means "all columns EXCEPT the last". So we get Age, Income, Spending Score as our feature matrix X. Everything the model uses as INPUT to make its prediction.
Part 2 Train the Model
🎓 PART 2 — Split data and train Logistic Regression model
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.25, random_state=2
)
# ↑ 75% training, 25% testing. random_state=2 = fixed split for reproducibility

LR = LogisticRegression()
# ↑ Create the Logistic Regression classifier object
#   Internally uses SIGMOID FUNCTION: maps any number → probability between 0 and 1
#   If probability > 0.5 → predict class 1 (YES)
#   If probability <= 0.5 → predict class 0 (NO)

LR.fit(X_train, y_train)
# ↑ Training! Model learns the DECISION BOUNDARY that separates class 0 and class 1
test_size=0.25, random_state=2
🔀 25% of data is kept for testing (model never sees this during training). random_state=2 seeds the randomness — same split every time you run the code.
LogisticRegression()
🔮 Despite the name "regression", this is a classifier. It uses the sigmoid function to output a probability (0 to 1), then converts it to a class (0 or 1) using a 0.5 threshold. Higher spending score + income → higher probability of purchase.
LR.fit(X_train, y_train)
📚 This single line does all the learning. The model adjusts its internal weights to find the best decision boundary separating buyers (1) from non-buyers (0).
Part 3 Confusion Matrix & Calculate All 5 Metrics
📊 PART 3 — Evaluate model performance with confusion matrix + 5 metrics
y_pred = LR.predict(X_test)
# ↑ Model predicts class (0 or 1) for each test sample

tn, fp, fn, tp = confusion_matrix(y_test, y_pred).ravel()
# ↑ confusion_matrix returns a 2×2 matrix:
#   [[TN, FP],    ← actual NO rows
#    [FN, TP]]    ← actual YES rows
# .ravel() FLATTENS it to [TN, FP, FN, TP] — 4 variables in one line!

accuracy    = (tn + tp) * 100 / (tp + tn + fp + fn)
precision   = tp / (tp + fp)
recall      = tp / (tp + fn)
f1_score    = (2 * precision * recall) / (precision + recall)
specificity = tn / (tn + fp)

print(f"Accuracy: {accuracy:.2f}%")     # {:.2f} = show 2 decimal places
print(f"Precision: {precision:.2f}")
print(f"Recall: {recall:.2f}")
print(f"F1 Score: {f1_score:.2f}")
print(f"Specificity: {specificity:.2f}")
LR.predict(X_test)
🔮 The trained model looks at 25% test data it's never seen before and predicts 0 or 1 for each customer. Returns an array like [0, 1, 1, 0, 1, ...]
confusion_matrix(y_test, y_pred).ravel()
🗂️ Compares actual labels (y_test) with predicted labels (y_pred) and builds the 2×2 table. .ravel() flattens [TN,FP; FN,TP] into a single list so we can unpack into 4 variables.
(tn + tp) * 100 / (tp+tn+fp+fn)
✅ Accuracy = (all correct predictions) ÷ (all predictions) × 100. Our model gets 96% — meaning it correctly classifies 96 out of 100 customers!
tp / (tp + fp)
🎯 Precision = out of everyone the model said "YES (will buy)", how many ACTUALLY bought? High precision = fewer false alarms.
tp / (tp + fn)
🔍 Recall = out of all customers who ACTUALLY bought, how many did the model catch? High recall = fewer missed cases. Critical in medical diagnosis!
2 * precision * recall / (precision + recall)
⚖️ F1 Score = harmonic mean of precision and recall. Single number that balances both. If either P or R is very low, F1 will be low too. Best overall metric.
tn / (tn + fp)
🔒 Specificity = out of all actual NO customers, how many did the model correctly say NO? Our 0.98 means the model is excellent at identifying non-buyers!

Part 4 Take User Input & Predict
🙋
WHY IS THIS CODE HERE?

This is the demo/interactive part — you type in feature values manually (like a new customer's age, income, spending score) and the model instantly predicts whether THAT customer will buy. It proves the model works on REAL new inputs, not just the test set.

🙋 PART 4 — Let user type feature values, then predict class
input_features = []
for i in range(X_train.shape[1]):
    # X_train.shape[1] = number of features (columns) = 3 in this case
    # Loop runs 3 times: once for each feature
    val = float(input(f"Feature {i+1}: "))
    # input() pauses and waits for user to type a value
    # float() converts the typed string to a decimal number
    input_features.append(val)

input_array = np.array(input_features).reshape(1, -1)
# np.array([2, 12, 45000]) → a 1D array
# .reshape(1, -1) → converts to 2D: [[2, 12, 45000]]
# sklearn's predict() needs 2D input (matrix, not a list)

prediction = LR.predict(input_array)[0]
# .predict() returns an array like [0] or [1]
# [0] extracts the first (and only) element from that array

print(f"Predicted Output: {prediction}")
# Output: 0 = customer will NOT buy, 1 = customer WILL buy
X_train.shape[1]
📐 .shape returns (rows, columns). [1] = columns = number of features. We loop this many times so the user enters one value per feature. No hardcoding!
float(input(...))
⌨️ input() = pause and wait for keyboard input. It always returns a string. float() converts that string "45000" into the number 45000.0 that the model can use.
np.array(...).reshape(1, -1)
🔄 The user inputs are a 1D list [2, 12, 45000]. sklearn's predict() needs a 2D matrix. reshape(1, -1) = "make it 1 row, and figure out the columns automatically (-1)". Result: [[2, 12, 45000]].
LR.predict(input_array)[0]
🎯 predict() always returns an array (even for one input). Since we only input one customer, we get [0] or [1]. The [0] at the end extracts just the prediction value.
